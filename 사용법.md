# grcwa 사용법

## 1. 기본 생성

```python
import torch
import grcwa

obj = grcwa.obj(
    101,
    [0.2, 0.0],
    [0.0, 0.2],
    1.0,
    torch.pi / 10,
    0.0,
    verbose=0,
    dtype_f=torch.float64,
    dtype_c=torch.complex128,
)
```

## 2. 레이어 추가

```python
obj.Add_LayerUniform(0.1, 1.0)
obj.Add_LayerGrid(0.2, 100, 100)
obj.Add_LayerUniform(0.1, 1.5)
```

모든 레이어를 추가한 뒤:

```python
obj.Init_Setup()
```

## 3. 입사파와 epsilon 입력

```python
obj.MakeExcitationPlanewave(1.0, 0.0, 0.0, 0.0, order=0, direction='forward')
obj.GridLayer_geteps(ep_grid)
```

grid layer 하나만 바꿀 때:

```python
obj.GridLayer_updateeps(which_layer, new_ep_grid, update_cache=False)
obj.UpdateSMatrixCache()
```

## 4. 기본 계산

```python
R, T = obj.RT_Solve(normalize=1)
ai, bi = obj.GetAmplitudes(which_layer, z_offset)
E, H = obj.Solve_FieldOnGrid(which_layer, z_offset)
```

`normalize=1`은 이제 고정된 `n/cos(theta)` 근사가 아니라 현재 excitation의
실제 incident flux로 나누므로, 입사 진폭을 바꾸거나 입사 매질에 loss가 있어도
더 안정적입니다.
`obj.normalization`에도 이와 같은 incident-power 값이 저장됩니다.

## 5. 캐시 사용

```python
obj.BuildSMatrixCache()
obj.BuildAmplitudeCache()
```

메모리가 중요하면:

```python
obj.ClearSMatrixCache()
```

이렇게 하면 무거운 prefix/suffix S-matrix는 지우고, 작은 amplitude/exterior cache만 남깁니다.

즉:

- `RT_Solve()`는 그대로 가능
- `GetAmplitudes*()`는 그대로 가능
- `Solve_FieldXY/XZ/YZ`도 그대로 가능

## 6. field API

기존 형식:

```python
E, H = obj.Solve_FieldOnGrid(which_layer, z_offset, components=('Ex', 'Hy'))
```

확장 형식:

```python
E, H = obj.Solve_FieldXY(which_layer, z_list, components=('Ex', 'Hy'))
E, H, x_coords, z_coords, layer_ranges, layer_edges, z_step = obj.Solve_FieldXZ(
    y0=0.0, znum=3, components=('Ex',)
)
E, H, y_coords, z_coords, layer_ranges, layer_edges, z_step = obj.Solve_FieldYZ(
    x0=0.0, znum=3, components=('Ex',)
)
```

추가 규칙:

- `Solve_FieldXZ()` / `Solve_FieldYZ()`는 전체 구조 단면
- `Solve_FieldXZLayer()` / `Solve_FieldYZLayer()`는 한 레이어 단면
- `Solve_FieldXY()`는 `E, H`를 반환하고 `E=[Ex,Ey,Ez]`, `H=[Hx,Hy,Hz]`
- 요청하지 않은 성분은 `None`
- 전기장만 요청하면 `H`는 `None`
- 정수 `znum`은 각 레이어 최소 샘플 수
- 두꺼운 레이어는 같은 `z_step` 기준으로 더 촘촘히 샘플
- `z_step=...`를 직접 주면 구조 전체 간격을 직접 제어 가능

## 7. save / load

가장 가벼운 baseline 저장:

```python
obj.SaveState('baseline.pt', include_caches=False)
restored = grcwa.obj.LoadState('baseline.pt')
```

가벼운 cached state 저장:

```python
obj.BuildSMatrixCache()
obj.BuildAmplitudeCache()
obj.ClearSMatrixCache()
obj.SaveState('baseline_light.pt', include_caches=True)
```

이 방식은 큰 S-matrix cache 없이 amplitude/exterior cache만 저장합니다.

## 8. 추천 워크플로

반복 field 조회:

```python
obj.BuildSMatrixCache()
obj.BuildAmplitudeCache()
obj.ClearSMatrixCache()
```

부분 구조 업데이트 반복:

```python
obj.GridLayer_updateeps(which_layer, new_ep_grid, update_cache=True)
```

또는

```python
obj.GridLayer_updateeps(which_layer, new_ep_grid, update_cache=False)
obj.UpdateSMatrixCache()
```

## 9. absorption

```python
abs_3d = obj.Solve_AbsorptionLayer(which_layer, [imag_eps_grid1, imag_eps_grid2], z_step=0.01, z_offset=0.0, z_min=5)
abs_z = obj.Solve_AbsorptionLayer(which_layer, [imag_eps_grid1, imag_eps_grid2], z_step=0.01, z_offset=0.0, z_min=5, avg='XY')
abs_xy = obj.Solve_AbsorptionLayer(which_layer, [imag_eps_grid1, imag_eps_grid2], z_step=0.01, z_offset=0.0, z_min=5, avg='Z')
layer_list = [1, 3]
pattern_list = [
    [layer1_imag_eps_a, layer1_imag_eps_b],  # layer 1용 pattern들
    [layer3_imag_eps_a],                     # layer 3용 pattern들
]
abs_all = obj.Solve_Absorption(layer_list, pattern_list, z_step=0.01, z_min=5, avg='XY')
abs_tot = obj.Solve_AbsorptionLayer(which_layer, [imag_eps_grid1, imag_eps_grid2], z_step=0.01, z_offset=0.0, z_min=5, avg='tot')
```

정리:

- 메인 함수는 `Solve_AbsorptionLayer()`
- `avg=None`이면 `(Nz, Nx, Ny)` absorption density를 유지
- `avg='XY'`면 `x,y`를 압축해서 `z` profile 반환
- `avg='Z'`면 `z`를 압축해서 `x,y` map 반환
- `avg='tot'`면 적분된 scalar 총흡수만 반환
- `pattern_absorption`, `pattern_total`은 입력한 pattern list 순서를 그대로 따름
- `Solve_Absorption(layer_list, pattern_list, ...)`에서 바깥 리스트는 `layer_list` 순서를 그대로 따르므로, 레이어마다 서로 다른 grid shape를 써도 됨

## 10. patterns / materials

`patterns/`로 epsilon grid를 만들어 바로 `GridLayer_geteps(...)`에 넣을 수 있습니다.

```python
from patterns import pattern_xx, nm
import torch

pat = pattern_xx(
    size_x=400 * nm,
    size_y=400 * nm,
    Nx=128,
    Ny=128,
    lamb0=1550 * nm,
    CRA=0.0,
    azimuth=0.0,
    eps_bg=1.0,
    eps_obj=3.48**2,
    radius=90 * nm,
    dtype_f=torch.float64,
    dtype_c=torch.complex128,
    device='cpu',
)

obj.GridLayer_geteps(pat.flatten())
obj.GridLayer_geteps(torch.cat([pat1.flatten(), pat2.flatten()]))
```

`materials/`는 `WL n k` 데이터를 읽어 `eps`까지 바로 가져옵니다.

```python
from materials import material
import torch

mat = material(
    1550e-9,
    dtype_f=torch.float64,
    dtype_c=torch.complex128,
    device='cpu',
)

eps_sio2 = mat.eps_SiO2_xx()
eps_old = mat.eps_SiO2_xx(filename='SiO2_index.txt')
```

기본 규칙:

- `WL`은 `nm`
- 코드에서 넣는 파장은 `m`
- `n`, `k`는 선형 보간 + 끝점 외삽
- `eps = (n + i k)^2`
- `SiO2_index.txt`와 dated 파일이 같이 있으면 최신 dated 파일을 기본 사용
